ITEM 3 — FEEDBACKS
Criar página desenvolvimento-feedbacks.tsx em /desenvolvimento/feedbacks.
Regras:

Gestor cria feedbacks para vendedores específicos ou para toda a equipe
Vendedor só vê feedbacks direcionados a ele ou para toda a equipe
Feedback tem: título, mensagem (texto rico), tipo (Elogio/Melhoria/Combinado/Aviso), data, autor
Vendedor pode marcar como "lido"
Gestor vê quem leu e quem não leu
Na sidebar, mostrar badge com número de feedbacks não lidos para o vendedor

Backend necessário:

GET /api/feedbacks — lista feedbacks do usuário (vendedor vê os seus, gestor vê todos)*
POST /api/feedbacks — cria feedback (somente gestor)*
PATCH /api/feedbacks/:id/lido — marca como lido*

Criar tabela feedbacks no banco:
sqlCREATE TABLE IF NOT EXISTS feedbacks (
  id SERIAL PRIMARY KEY,
  tenant_id INTEGER NOT NULL REFERENCES tenants(id),
  autor_id INTEGER NOT NULL REFERENCES users(id),
  destinatario_id INTEGER REFERENCES users(id), -- null = toda equipe
  titulo VARCHAR(200) NOT NULL,
  mensagem TEXT NOT NULL,
  tipo VARCHAR(20) NOT NULL DEFAULT 'combinado',
  lido_por JSONB DEFAULT '[]',
  created_at TIMESTAMP NOT NULL DEFAULT NOW()
);"*
