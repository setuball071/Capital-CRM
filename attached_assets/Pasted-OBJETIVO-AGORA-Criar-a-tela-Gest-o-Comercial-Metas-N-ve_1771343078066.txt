Precisamos desenvolver o módulo completo de configuração de Metas & Níveis dentro de:

ALPHA → Gestão Comercial → Metas & Níveis

Objetivo:
Permitir que o gestor configure níveis de premiação para dois tipos de produção:
- PRODUÇÃO GERAL
- PRODUÇÃO CARTÃO

⚙️ Estrutura técnica:

Criar tabela production_levels com os seguintes campos:

id (uuid)
tenant_id
type ("GERAL" | "CARTAO")
name (string)
minimum_amount (decimal)
prize_amount (decimal)
icon (string)
color (string hex)
order_position (integer)
active (boolean)
created_at (timestamp)

📌 Interface:

A tela deve ter duas abas:
[ Produção Geral ]
[ Produção Cartão ]

Cada aba exibe uma lista de níveis com:

- Ordem
- Ícone
- Nome do nível
- Faturamento mínimo
- Valor do prêmio
- Cor
- Ações (editar / excluir)

Botão:
+ Adicionar nível

📌 Modal de criação/edição:

Campos:
- Nome do nível
- Faturamento mínimo (R$)
- Valor do prêmio (R$)
- Selecionar ícone (dropdown com preview)
- Selecionar cor (color picker)
- Ordem
- Ativo (toggle)

🎨 Ícones disponíveis (usar biblioteca de ícones existente no projeto):

gem
trophy
medal
star
crown
target
shield
award
flame
rocket

📌 Regras:

- Sempre ordenar por minimum_amount ASC.
- O sistema deve permitir múltiplos níveis.
- Não pode permitir dois níveis com mesmo minimum_amount no mesmo type.
- O tipo GERAL é independente do tipo CARTAO.

📌 Importante:

Esse módulo será usado depois para:
- Determinar nível atual do vendedor
- Determinar próximo nível mesmo quando não houver nível atingido
- Calcular quanto falta para próximo nível

Implementar API REST completa:
GET /levels
POST /levels
PUT /levels/:id
DELETE /levels/:id
