# PROMPT — Integração Webhook: Etiquetas WhatsApp → CRM Capital Go

## OBJETIVO

Receber eventos de etiquetagem da plataforma de WhatsApp e aplicar
as etiquetas nos clientes correspondentes do CRM, usando CPF como chave de ligação.
Adicionar também o botão de acesso à plataforma de WhatsApp na interface do CRM.

## T002 — Endpoint webhook: POST /api/webhooks/whatsapp-labels

Criar endpoint **público** (sem autenticação de usuário, mas com validação de secret):

```ts
POST /api/webhooks/whatsapp-labels
```

### Headers esperados:
```
x-webhook-secret: {WHATSAPP_WEBHOOK_SECRET}   // variável de ambiente
Content-Type: application/json
```

### Body recebido (enviado pela plataforma de WhatsApp):
```json
{
  "event": "label.applied",
  "label": {
    "id": "abc123",
    "name": "Compra Dívida",
    "color": "#6C2BD9"
  },
  "contact": {
    "phone_number": "5548999999999",
    "name": "Marta Alves Vieira",
    "cpf": "316.298.903-59"
  },
  "conversation_id": "conv_xyz",
  "timestamp": "2025-06-01T14:30:00Z"
}
```

### Eventos suportados:
- `label.applied` → adicionar etiqueta ao cliente
- `label.removed` → remover etiqueta do cliente

### Lógica do endpoint:

```ts
// 1. Validar secret
const secret = req.headers['x-webhook-secret'];
if (secret !== process.env.WHATSAPP_WEBHOOK_SECRET) {
  return res.status(401).json({ error: 'Unauthorized' });
}

// 2. Extrair CPF do body (limpar máscara)
const cpfLimpo = body.contact.cpf.replace(/\D/g, '');

// 3. Buscar cliente no CRM pelo CPF (tenant-agnostic neste endpoint,
//    ou usar tenant fixo se toda a plataforma é single-tenant por ora)
const cliente = await db.query.clients.findFirst({
  where: eq(clients.cpf, cpfLimpo)  // ajustar nome da coluna conforme schema atual
});

if (!cliente) {
  // Logar tentativa sem match para auditoria
  console.warn(`[webhook] Cliente não encontrado para CPF: ${cpfLimpo}`);
  return res.status(200).json({ 
    status: 'not_found', 
    message: 'Cliente não encontrado no CRM' 
  });
  // Retorna 200 para a plataforma não retentar indefinidamente
}

// 4. Atualizar labels
const labelsAtuais: string[] = cliente.whatsappLabels || [];
const nomeEtiqueta = body.label.name;

let labelsAtualizadas: string[];

if (body.event === 'label.applied') {
  // Adicionar se não existir
  labelsAtualizadas = labelsAtuais.includes(nomeEtiqueta)
    ? labelsAtuais
    : [...labelsAtuais, nomeEtiqueta];
} else if (body.event === 'label.removed') {
  // Remover
  labelsAtualizadas = labelsAtuais.filter(l => l !== nomeEtiqueta);
} else {
  return res.status(200).json({ status: 'ignored', event: body.event });
}

// 5. Salvar no banco
await db.update(clients)
  .set({ 
    whatsappLabels: labelsAtualizadas,
    updatedAt: new Date()
  })
  .where(eq(clients.id, cliente.id));

// 6. Retornar sucesso
return res.status(200).json({
  status: 'success',
  clienteId: cliente.id,
  labels: labelsAtualizadas
});
```

### Variável de ambiente necessária:
Adicionar ao `.env` e ao Replit Secrets:
```
WHATSAPP_WEBHOOK_SECRET=capitalgo_whatsapp_secret_2025
```

## T003 — Log de eventos webhook (tabela de auditoria)

Criar tabela `webhook_logs` para rastrear todos os eventos recebidos:

```ts
export const webhookLogs = pgTable('webhook_logs', {
  id:           serial('id').primaryKey(),
  source:       text('source').notNull(),        // 'whatsapp'
  event:        text('event').notNull(),          // 'label.applied' | 'label.removed'
  cpf:          text('cpf'),                      // CPF recebido (sem máscara)
  clienteId:    integer('cliente_id'),            // null se não encontrado
  labelName:    text('label_name'),
  payload:      jsonb('payload'),                 // body completo recebido
  status:       text('status').notNull(),         // 'success' | 'not_found' | 'error'
  errorMessage: text('error_message'),
  createdAt:    timestamp('created_at').defaultNow(),
});
```

Registrar cada chamada ao webhook nessa tabela (success ou failure).

## RESUMO TÉCNICO

| Item | Detalhe |
|------|---------|
| Chave de ligação | CPF (sem máscara nos dois lados) |
| Autenticação webhook | Header `x-webhook-secret` + env var |
| Eventos suportados | `label.applied`, `label.removed` |
| Retorno em caso de CPF não encontrado | HTTP 200 (evita retry loop) |
| Auditoria | Tabela `webhook_logs` com payload completo |
| Botão WhatsApp | Header do CRM, abre em nova aba |
| Env vars necessárias | `WHATSAPP_WEBHOOK_SECRET`, `WHATSAPP_PLATFORM_URL`, `VITE_WHATSAPP_PLATFORM_URL` |
