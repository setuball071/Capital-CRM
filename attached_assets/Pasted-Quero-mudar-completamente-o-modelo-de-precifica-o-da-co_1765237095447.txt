Quero mudar completamente o modelo de precificação da compra de lista.

Em vez de calcular preço por registro via interpolação, quero usar um modelo de PACOTES com faixas fixas.

## 1) Tabela de pacotes (config inicial)

Crie uma tabela de configuração (no banco ou em JSON) com algo assim:

[
  { "quantidade_maxima": 100,    "nome_pacote": "Pacote 100",    "preco": 37.90 },
  { "quantidade_maxima": 300,    "nome_pacote": "Pacote 300",    "preco": 67.90 },
  { "quantidade_maxima": 500,    "nome_pacote": "Pacote 500",    "preco": 97.90 },
  { "quantidade_maxima": 1000,   "nome_pacote": "Pacote 1000",   "preco": 187.90 },
  { "quantidade_maxima": 2000,   "nome_pacote": "Pacote 2000",   "preco": 297.90 },
  { "quantidade_maxima": 3000,   "nome_pacote": "Pacote 3000",   "preco": 397.90 },
  { "quantidade_maxima": 5000,   "nome_pacote": "Pacote 5000",   "preco": 597.90 },
  { "quantidade_maxima": 10000,  "nome_pacote": "Pacote 10000",  "preco": 997.90 }
]

Depois quero poder editar isso via tela de Config. Preços, mas por enquanto pode deixar fixo em config.

## 2) Regra de cálculo

Dado um pedido com quantidade real de registros Q:

1. Se Q <= 0: não permite gerar pedido.
2. Encontra o primeiro pacote onde Q <= quantidade_maxima.
3. Usa o preço desse pacote como preco_total e registra também:
   - nome_pacote
   - quantidade_maxima_do_pacote
4. Se Q for maior que o maior pacote configurado, repetir o maior pacote “em blocos” ou simplesmente aplicar o maior pacote uma única vez (pode deixar comentado esse comportamento para eu ajustar depois).

## 3) Integração na tela de Compra de Lista

- Ao simular os filtros, contar Q (quantidade retornada).
- Chamar um endpoint de simulação, por exemplo: GET /api/pedidos/simular-preco?quantidade=Q
- Esse endpoint retorna:

{
  "quantidade_real": Q,
  "nome_pacote": "Pacote 2000",
  "quantidade_pacote": 2000,
  "preco_total": 297.90
}

- O botão "Gerar Pedido de Lista" deve mostrar:

  Gerar Pedido de Lista (Q registros – Pacote 2000 – R$ 297,90)

## 4) Consistência

- O mesmo cálculo deve ser usado:
  - na simulação,
  - ao salvar o pedido,
  - ao aprovar o pedido no painel do admin.

- A tela de "Config. Preços" pode ser adaptada depois para editar essa tabela de pacotes.
