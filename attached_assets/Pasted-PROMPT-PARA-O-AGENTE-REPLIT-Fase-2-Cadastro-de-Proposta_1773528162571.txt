PARTE A — ROTAS DA API
Criar em server/routes.ts (ou arquivo separado server/contracts.ts importado no routes principal):
Fluxos (master only)
GET    /api/contracts/flows              — listar fluxos do tenant
POST   /api/contracts/flows              — criar fluxo
PUT    /api/contracts/flows/:id          — editar fluxo
DELETE /api/contracts/flows/:id          — remover fluxo
GET    /api/contracts/flows/:id/steps    — listar etapas do fluxo
POST   /api/contracts/flows/:id/steps    — adicionar etapa
PUT    /api/contracts/flows/steps/:id    — editar etapa
DELETE /api/contracts/flows/steps/:id    — remover etapa
Propostas
GET  /api/contracts/proposals            — listar (filtra por role: vendedor vê só as suas)
POST /api/contracts/proposals            — criar proposta (vendedor, coordenacao, master)
GET  /api/contracts/proposals/:id        — detalhe da proposta
PUT  /api/contracts/proposals/:id/status — avançar status / registrar ADE (operacional+)
POST /api/contracts/proposals/:id/pause  — pendenciar para corretor ou banco
POST /api/contracts/proposals/:id/resume — retomar após pendência
GET  /api/contracts/proposals/:id/history   — histórico de movimentações
GET  /api/contracts/proposals/:id/messages  — mensagens da proposta
POST /api/contracts/proposals/:id/messages  — enviar mensagem + upload de documento
Grupos de comissão (master/financeiro)
GET  /api/contracts/commission-groups    — listar grupos
POST /api/contracts/commission-groups    — criar grupo
PUT  /api/contracts/commission-groups/:id
DELETE /api/contracts/commission-groups/:id
Regras de negócio nas rotas:

Toda rota verifica tenant_id do usuário logado
vendedor só acessa propostas onde vendor_id = currentUser.id
operacional acessa todas as propostas do tenant
Ao criar proposta: buscar fluxo ativo que bata com bank + convenio + product, atribuir automaticamente. Se não encontrar fluxo, criar proposta sem fluxo vinculado
Ao avançar status: registrar em proposal_history automaticamente
Ao pausar: setar is_paused = true e paused_at_step_id = current_step_id
Ao retomar: setar is_paused = false, paused_at_step_id = null, manter na mesma etapa
