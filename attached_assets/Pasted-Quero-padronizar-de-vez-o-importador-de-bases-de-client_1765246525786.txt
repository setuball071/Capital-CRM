### 2. COMO O IMPORTADOR DEVE FUNCIONAR

2.1. Ler sempre o HEADER da planilha
- Na primeira linha da planilha, leia todos os cabeçalhos.
- Normalize: transforme em minúsculas e remova espaços/acentos se precisar.
- Monte um mapa: { "cpf": índice, "matricula": índice, "banco_emprestimo": índice, ... }

2.2. Nunca usar índice “na mão”
- Nada de código tipo `row.getCell(5)` assumindo que a coluna 5 é banco.
- Sempre usar o mapa de header:
  - `const cpf = row.getCell(headers["cpf"]).value` (se existir)
  - `const bancoEmprestimo = row.getCell(headers["banco_emprestimo"]).value` (se existir)

2.3. Mapear cada grupo para as tabelas certas

[CLIENTE (tabela de pessoas/base)]
- cpf
- matricula
- convenio
- orgao
- uf
- municipio
- situacao_funcional
- data_nascimento
- telefone_1, telefone_2, telefone_3, email
- banco_salario, agencia_salario, conta_salario

[FOLHA / MARGENS]
- competencia_folha
- margens de 70, 35, cartão crédito, cartão benefício (bruta/utilizada/saldo)

Importante: para SIAPE, a ordem de exibição que vamos usar depois é:
- 70% (compulsória)
- 35% consignado
- 5% cartão crédito
- 5% cartão benefício

Mas aqui na importação, basta salvar os campos corretos.

[CONTRATOS]
- banco_emprestimo  -> deve ser salvo como o NOME DO BANCO DO EMPRÉSTIMO
- tipo_produto
- valor_parcela
- saldo_devedor
- prazo_remanescente  -> salvar exatamente esse valor, sem cálculos
- competencia_contrato
- numero_contrato
- situacao_contrato

2.4. Campos opcionais
- Se alguma coluna do template não existir na planilha, ignore silenciosamente.
- Não quebre a importação se faltar telefone_2, por exemplo.
- O que não veio, fica null/undefined na base.

### 3. REGRA ESPECIAL PRA BANCO

Garanta no código que:

- Salvar este valor em contratos.parcelas_restantes.
- Não calcular prazo total, não inferir parcelas pagas, nada disso.
