12 - Fix D8 Import Header Validation
What & Why
The import service that runs in production is FastImportService in server/fast-import-service.ts. The validation logic inside loadToStaging() throws a hard error when any of the "required" D8 headers are missing. Previous fixes were applied to streaming-import-service.ts, but that file is NOT used for D8 imports in production — fast-import-service.ts is.

The uploaded file ALFA FI SERV d8_d8_organizado.csv consistently triggers the error: Headers obrigatórios D8 Servidor ausentes: MATRICULA, TIPO_CONTRATO, SITUACAO_CONTRATO — even though visually those columns appear to be present. The root cause is likely:

The column name map for D8_COLUMN_MAP_SERVIDOR is missing common aliases/variations used by different bank exports.
The validation uses a hard error (throws) instead of allowing the import to proceed and surfacing column issues as row-level errors.
Done looks like
Uploading a D8 Servidor file with headers BANCO;ORGAO;MATRICULA;UF;NOME;CPF;TIPO_CONTRATO;PMT;PRAZO_REMANESCENTE;SITUACAO_CONTRATO;NUMERO_CONTRATO no longer throws the validation error.
The import proceeds and processes rows, either succeeding or producing per-row errors.
The same fix is applied to the D8 Pensionista header validation.
The competencia field is NOT required for D8 imports (it can be absent without error).
Out of scope
Changes to streaming-import-service.ts (already done, wrong file)
New D8 column types or new import types
Tasks
Expand D8 column aliases — In D8_COLUMN_MAP_SERVIDOR, add additional key aliases for matricula (e.g., mat, nr_matricula, num_matricula), tipo_contrato (e.g., tipo, contrato_tipo, produto), situacao_contrato (e.g., situacao, sit_contrato, status_contrato, status). Do the same for D8_COLUMN_MAP_PENSIONISTA.

Change D8 validation from hard-error to soft-warning — Instead of throwing an error when optional headers are missing, log a warning and allow the import to proceed. Only cpf should remain a hard requirement (imports cannot proceed without CPF). For all other "required" headers, treat absence as a soft warning, and rows missing those values will simply have null in those columns.

Add encoding-aware header reading — In readHeaders(), detect if the file is Windows-1252 encoded (presence of bytes > 0x7F not matching valid UTF-8 sequences) and re-read using latin1 encoding. This ensures headers like SITUAÇÃO and others with accents are decoded correctly regardless of source encoding. (Reuse the same pattern from streaming-import-service.ts if present, or implement via a two-pass approach: try UTF-8, if header count is 1 and line looks wrong try latin1.)
