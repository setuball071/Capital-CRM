Contexto
Projeto Node + TypeScript rodando no Replit.
Backend em /server.
Já existe importação antiga funcionando, mas não escala para arquivos com milhões de linhas.

Precisamos adaptar o sistema para importações massivas (10+ milhões de linhas), mantendo o modelo antigo funcional.

Arquivos principais envolvidos:

server/import-service.ts

server/routes.ts

server/storage.ts

server/tenant-middleware.ts

OBJETIVO GERAL

Criar um novo pipeline de importação em JOB + STREAM, capaz de importar até 1.000.000 de linhas por execução, com retomada automática, logs de erro por linha, e 3 tipos distintos de importação, sem carregar o arquivo inteiro em memória.

TIPOS DE IMPORTAÇÃO (OBRIGATÓRIO)

A importação SEMPRE acontece em 3 camadas separadas:

1) FOLHA / FINANCEIRO (mensal)

Arquivo com dados financeiros do cliente.

Balisadores:

CPF + MATRÍCULA + COMPETÊNCIA

A competência será informada manualmente pelo usuário no upload.

Atualiza:

pessoa

clientes_vinculo

mes (fotografia mensal)

2) BANCOS / D8 (por banco)

Arquivo com descontos bancários.

Balisadores:

Cruzamento cliente: CPF + MATRÍCULA

Âncora do contrato: BANCO + NUMERO_CONTRATO

Banco será informado manualmente no upload.

Pode ter layout SERVIDOR ou PENSIONISTA.

Pensionista traz M_INSTITUIDOR, que deve ser salvo em extras (json).

3) CONTATOS / HIGIENIZAÇÃO

Arquivo com telefones, emails e endereço.

Balisador:

CPF apenas

Telefones/emails deduplicados.

Contato vale para todas as matrículas do CPF.

REGRAS TÉCNICAS OBRIGATÓRIAS
Tipos e preservação de dados

CPF, MATRÍCULA, UPAG, NUMERO_CONTRATO:

armazenar como TEXT

apenas dígitos

aplicar padStart

PMT:

armazenar como numeric

exportar normal para Excel

Nunca permitir notação científica.

ESCALA (PONTO CRÍTICO)

NÃO ler arquivo inteiro em memória.

Usar streaming (fs.createReadStream + readline ou parser CSV em stream).

Processar em buffers internos (100k–200k linhas).

Cada execução do job processa até 1.000.000 de linhas.

Ao atingir 1M:

salvar progresso

encerrar execução

permitir retomada manual ou automática.
