### 2. Serviço de Prompt (server/services/creativePromptService.ts)

Função `buildImagePrompt(formData)` que recebe:
- `theme`: string (ex: "portabilidade consignado", "refinanciamento")
- `convenio`: string (ex: "SIAPE", "INSS", "Governo Estadual - SC")
- `formato`: enum 'square' | 'portrait' | 'story' (1:1, 9:16, 4:5)
- `headline`: string (frase principal do criativo)
- `examples`: array de até 3 objetos { label, value } (ex: {label: "Parcela", value: "R$ 320/mês"})
- `cta`: string (call to action, ex: "Solicite uma simulação agora")
- `style`: enum 'fotorrealista' | 'moderno_clean' | 'bold_typographic'

E retorna um prompt em inglês, otimizado para Imagen 4, seguindo este template:
Professional financial services advertisement for Brazilian consignado credit company.
[style_instruction].
Color palette: deep purple (#6C2BD9), electric blue (#1E88E5), vibrant pink (#E91E63) as accents on dark background.
Logo area reserved at top center.
Main headline in large bold white text: "[headline]"
Subtitle: "[convenio] - [theme]"
[examples_block: se houver, mostrar em cards/badges dentro do criativo]
CTA button: "[cta]"
Bottom area: clean space for agent signature.
Style: [fotorrealista = cinematic lighting, real people, professional photography style | moderno_clean = flat design, geometric shapes, minimal | bold_typographic = typography-focused, strong contrast, editorial]
High quality, marketing material, no watermarks, no text errors.

### 3. Serviço de Geração (server/services/imagenService.ts)

Usar a Gemini API (Google AI Studio) com o modelo `imagen-4.0-generate-001`.

Variável de ambiente necessária: `GOOGLE_AI_API_KEY`

Endpoint: `POST https://generativelanguage.googleapis.com/v1beta/models/imagen-4.0-generate-001:generateImages?key={API_KEY}`

Body:
```json
{
  "prompt": { "text": "<prompt gerado>" },
  "imageCount": 4,
  "aspectRatio": "<1:1 | 9:16 | 4:5 conforme formato>",
  "safetyFilterLevel": "BLOCK_MEDIUM_AND_ABOVE"
}
```

A resposta retorna base64. Salvar as imagens no próprio banco como data URLs (temporário) ou fazer upload para storage se já existir no sistema.

---

### 4. Rate Limiting (middleware)

Criar middleware `checkCreativeQuota` que:
1. Lê `creative_generation_quota` para (user_id, hoje)
2. Se count >= 5 → retorna 429 com mensagem: "Você atingiu o limite de 5 criações por dia. Tente novamente amanhã."
3. Se count < 5 → deixa passar e incrementa o counter após geração bem-sucedida
4. Se não existe registro → cria com count = 1

---

### 5. Rotas (server/routes/creatives.ts)

`POST /api/creatives/generate`
- Auth: todos os roles
- Middleware: `checkCreativeQuota`
- Body: formData (theme, convenio, formato, headline, examples, cta, style)
- Chama `buildImagePrompt` → `imagenService.generate` → salva na tabela
- Retorna: { generationId, images: [url1, url2, url3, url4], quotaUsed: N, quotaLimit: 5 }

`POST /api/creatives/save`
- Auth: todos os roles
- Body: { generationId, selectedImageUrl, name, pack }
- Salva o criativo selecionado na tabela principal de criativos
- Marca a geração como status = 'saved'

`GET /api/creatives/quota`
- Retorna: { used: N, limit: 5, resetsAt: "amanhã às 00:00" }
