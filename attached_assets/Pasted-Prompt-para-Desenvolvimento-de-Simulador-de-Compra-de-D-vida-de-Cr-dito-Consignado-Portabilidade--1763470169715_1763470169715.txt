Prompt para Desenvolvimento de Simulador de Compra de Dívida de Crédito Consignado (Portabilidade)🎯 ObjetivoDesenvolver um Web App (Single-Page Application) em HTML, CSS (Tailwind CSS) e JavaScript puro, rodando no Replit, que permita ao usuário simular a Compra de Dívida (Portabilidade) de um Contrato de Crédito Consignado existente para uma nova instituição.⚙️ Funcionalidades e Requisitos TécnicosEstrutura da Aplicação:Tecnologia: HTML, JavaScript puro (ES6+), e Tailwind CSS (via CDN) para estilização.Responsividade: O layout deve ser totalmente responsivo (desktop, tablet, mobile).Entrada de Dados (Contrato Atual):O usuário deve inserir os seguintes dados do seu contrato de dívida consignada atual:Saldo Devedor Atual (R$): Valor principal remanescente.Prazo Restante (Meses): Número de parcelas que faltam.Valor da Parcela Atual (R$): Valor pago mensalmente.Taxa de Juros Efetiva Anual (%): Taxa de juros do contrato atual (para fins de validação e comparação).Entrada de Dados (Nova Proposta - Simulação):O usuário deve inserir os seguintes dados da nova proposta de portabilidade:Nova Taxa de Juros Efetiva Anual (%): Taxa oferecida pela nova instituição.Novo Prazo Total Desejado (Meses): Prazo total para o novo contrato (não pode ser menor que o prazo restante).Cálculos e Lógica de Negócio:A simulação deve calcular os seguintes valores, utilizando a Fórmula de Amortização Price (Tabela Price):Cálculo A: Valor Presente (VP) do Saldo Devedor Atual:Confirmar que o Saldo Devedor Atual inserido corresponde ao VP das parcelas restantes, usando a Taxa Atual e o Prazo Restante. Isso serve como validação dos dados de entrada.Cálculo B: Nova Parcela (NP):Calcular o novo valor da parcela (NP) para o Saldo Devedor Atual (VP) utilizando a Nova Taxa de Juros e o Novo Prazo Total.Fórmula Price (Amortização Constante): $P = VP \cdot \frac{i}{1 - (1+i)^{-n}}$Onde: $P$ é a Parcela, $VP$ é o Saldo Devedor, $i$ é a taxa de juros mensal (Nova Taxa / 1200), e $n$ é o Novo Prazo Total.Cálculo C: Troco/Diferença (Liberação de Saldo):Troco (Valor Líquido Recebido): Calcular a diferença entre o Valor da Parcela Atual e a Nova Parcela (NP). Neste cenário focado em portabilidade com liberação de saldo (parcela menor), o foco é a redução da parcela, não o Troco. No entanto, o campo Troco é comum em refinanciamentos/portabilidade. Vamos focar na redução da parcela.Redução da Parcela Mensal: $Redução = Parcela\ Atual - Nova\ Parcela$Economia Total: $Economia = Redução \times Novo\ Prazo\ Total$ (Ignorando valor temporal do dinheiro, apenas para um indicador simples).Exibição de Resultados (Dashboard):A aplicação deve exibir os resultados de forma clara em um "dashboard de comparação" após o cálculo:Contrato Atual:Saldo Devedor: R$ XXXX,XXParcela Atual: R$ YYY,YYPrazo Restante: ZZ mesesNova Proposta (Portabilidade):Nova Taxa Efetiva Anual: X,XX%Nova Parcela Mensal: R$ AAA,AANovo Prazo Total: BBB mesesDestaque (Métricas Chave):Redução Mensal na Parcela: R$ CCC,CC (e percentual de redução)Economia Total Estimada: R$ DDDD,DD🎨 Design e UI/UX (Tailwind CSS)Estilo: Use classes do Tailwind para criar um design moderno, limpo e com foco em acessibilidade.Elementos: Utilize inputs bem rotulados, botões de ação (Simular), e cartões (cards) para exibir os resultados.Aparência: Cores primárias de confiança (azul/verde), fontes legíveis (sans-serif), e cantos arredondados (rounded-lg).Feedback: Adicionar um feedback visual (loading/mensagem de erro) durante a simulação ou se os dados de entrada forem inválidos.📝 Estrutura de Arquivos (Replit - Single File)O projeto será um único arquivo index.html contendo todo o HTML, CSS (via CDN) e JavaScript.
